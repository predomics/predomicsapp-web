1. Project Overview

What is PredomicsApp?

PredomicsApp is a production-grade, full-stack web application that provides an interactive interface for gpredomics, a Rust-accelerated engine for building interpretable predictive models from high-dimensional omics data (metagenomics, metabolomics, transcriptomics).

The application implements sparse discrete linear classifiers — models that use only a handful of features with integer coefficients (+1, -1, +2, etc.) — making them inherently interpretable and deployable as simple biomarker signatures. The optimization is performed via genetic algorithms (GA), beam search, or MCMC, producing a population of near-optimal models rather than a single solution.

Key Innovation: Rather than returning a single model, PredomicsApp explores the Family of Best Models (FBM) — the statistically defined population of all models whose performance is not significantly worse than the best. This enables ecosystem-level analysis of feature co-selection, stability, and biological relevance.

ML Engine — gpredomicspy

The core ML engine is gpredomicspy, a hybrid Rust + Python package that implements:

  • Feature filtering: Rust-accelerated prevalence and significance filtering (Wilcoxon, Student's t, Bayesian Fisher)
  • Optimization: Genetic Algorithm (GA), Beam Search, MCMC for sparse model search
  • Model languages: Binary (±1), Ternary (0,±1), Ratio, Power-of-2 encodings
  • Cross-validation: Stratified k-fold with train/test tracking
  • Evaluation: AUC, accuracy, sensitivity, specificity, fit (composite metric)

8. Algorithms & Methods

Model Languages

PredomicsApp implements sparse discrete linear classifiers with four encoding schemes:

LanguageCoefficientsFormulaInterpretability
Binary+1, -1score = Σ(±1 × feature_i)Highest — simple presence/absence voting
Ternary0, +1, -1score = Σ(coef × feature_i)High — allows feature exclusion
Ratiolog-ratio pairsscore = Σ log(feature_i / feature_j)Medium — relative abundance ratios
Power-of-2±1, ±2, ±4...score = Σ(2^k × feature_i)Medium — integer-weighted linear sum

Optimization Algorithms

  • Genetic Algorithm (GA): Population-based evolutionary search with crossover, mutation, and tournament selection. Default method with best exploration of the model space.
  • Beam Search: Deterministic breadth-first search expanding top-k candidates at each sparsity level. Faster but less exploratory.
  • MCMC (Markov Chain Monte Carlo): Stochastic sampling of the model space with Metropolis-Hastings acceptance criterion.

Family of Best Models (FBM) & Stability

The FBM is the statistically defined population of all models whose fitness is not significantly worse than the best. Two CI methods are available:

  • Standard Wald CI: threshold = r - z × √(r(1-r)/n)
  • Blaise CI: threshold = r - (0.5/n + z × √(r(1-r)/n)) — adds continuity correction

Stability is measured across sparsity levels using:

  • Kuncheva index: Pairwise overlap consistency of feature sets
  • Tanimoto similarity: Jaccard-like coefficient for feature set comparison
  • CW_rel: Relative consistency-weighted index

Ecosystem & Network Analysis

The co-abundance network analysis pipeline:

  1. Prevalence filtering: Remove features below threshold (default 30%)
  2. Spearman correlations: Matrix-wise computation (O(n²) for feature pairs)
  3. Edge filtering: Keep pairs with |ρ| ≥ threshold (default 0.3)
  4. Community detection: Louvain algorithm (NetworkX, seed=42 for reproducibility)
  5. Node metrics: Degree, betweenness centrality, class enrichment
  6. Taxonomy annotation: MSP lookup from biobanks.gmt.bio with hierarchical coloring
  7. Co-presence analysis: Hypergeometric test for pairwise feature co-occurrence in FBM

Plotly.js Integration

All charts use Plotly.js (lazy-loaded for performance) with consistent theming via the useChartTheme composable. Charts are rendered with { responsive: true, displayModeBar: false } and support dark/light mode switching via a watch on themeStore.isDark.

Dark/Light Theming

The chartColors() function returns a theme-aware color palette with properties: text, grid, paper, plot, positive (deepskyblue), negative (firebrick), neutral (darkorchid), and alpha variants. The chartLayout() function applies these colors to a Plotly layout object.

10. Deployment & Configuration

Docker

Multi-stage Dockerfile:

# Stage 1: Build frontend (Node.js)
FROM node:20-alpine AS frontend-build
WORKDIR /app/frontend
RUN npm ci && npm run build

# Stage 2: Backend (Python)
FROM python:3.11-slim
COPY --from=frontend-build /app/frontend/dist /app/static
RUN pip install -e .
CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]

Kubernetes (Helm)

Helm charts in helm/ provide production Kubernetes deployment with: configurable replicas, resource limits, PostgreSQL StatefulSet, Ingress with TLS, PersistentVolumeClaim for data storage, and health check probes.

11. Testing

Backend Tests (pytest)

215+ test cases using pytest and pytest-asyncio with an in-memory SQLite database. Tests cover API endpoints, authentication flows, authorization checks, data processing, export generation, and error handling. Run with: 

cd backend && python -m pytest --cov=app --cov-report=term-missing

cd frontend && npx vitest run --coverage
